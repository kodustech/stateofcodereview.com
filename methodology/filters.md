# Filters and scope

## Engine cutoff: 2025-09-01

Kodus introduced a new label taxonomy in September 2025. The current taxonomy uses 5 canonical labels:

- `bug`
- `security`
- `performance`
- `cross-file`
- `kody rules`

Suggestions generated before this cutoff used a different label set (error_handling, potential_issues, maintainability, refactoring, documentation_and_comments, code_style, performance_and_optimization). Those legacy labels were generated by a different LLM prompt and pipeline. To keep findings interpretable as "current state of AI code review," all pre-2025-09-01 suggestions are excluded from analysis.

## Label whitelist

Analysis includes only suggestions where `label IN ('bug', 'security', 'performance', 'cross_file', 'cross-file')`. The KodyRules census (Chapter 4) draws from a separate table that is unaffected by engine versioning.

## Privacy thresholds

- No slice with fewer than 50 suggestions is published as a finding.
- Programming language slices with fewer than 5 distinct organizations are footnoted as low-confidence.
- Organization, repository, and pull request identifiers are never exposed.
- Suggestion text shown in the report is reviewed for sensitive substrings before publication.

## What is not included

- Suggestions from Kodus self-hosted deployments (the data warehouse covers Cloud only).
- Suggestions without explicit developer feedback. Unobserved suggestions are excluded because reception cannot be measured.
- Pull requests from organizations that opted out of research (reserved as policy; none currently).
