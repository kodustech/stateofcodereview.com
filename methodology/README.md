# Methodology

How the data was produced.

## Source

All data is sourced from the Kodus production code review pipeline. Kodus is an open source AI code review tool that reviews pull requests across GitHub, GitLab, Bitbucket, and Azure DevOps.

The full dataset lives in BigQuery project `kody-408918`. Tables used:

- `kodus_postgres.suggestion_embedded` — per-suggestion record with developer feedback
- `kodus_mongo.pullRequests` — pull-request-level metadata and raw suggestion funnel
- `kodus_mongo.kodyRules` — custom review rules codified by each organization
- `kodus_postgres.organizations` — organization metadata (for distinct counts only)

## Filters

Three filters define what enters the report. See [filters.md](./filters.md) for full details.

1. **Engine cutoff:** only suggestions with `createdAt >= '2025-09-01'`, the date the current label taxonomy stabilized.
2. **Label whitelist:** only suggestions with labels `bug`, `security`, `performance`, `cross-file`. Legacy labels are excluded because they were generated by a deprecated engine version.
3. **Privacy threshold:** no slice with fewer than 50 suggestions is published. Organization, repository, and pull request identifiers are never exposed.

## Reproducibility

Every chart in the report has a backing SQL query in [queries/](./queries/). Queries are in BigQuery Standard SQL dialect.

External researchers cannot run these queries directly. To validate findings, contact research@kodus.io for a controlled dataset extract under data use agreement.

## Consent and ethics

Organizations using Kodus consent to aggregated, anonymized research per the Terms of Service. No individually identifiable data is published. Suggestion text shown in the report is screened for sensitive substrings (internal hostnames, project names, customer identifiers) before publication.
